2. DNS (Domain Name System, система доменных имен) - база данных, сопоставляющая доменное имя сайта с его числовым адресом.
Числовой адрес сайта представляет собой комбинацию из 4 чисел от 0 до 255.
DNS состоит из протокола и DNS-сервера. Через протокол сайт обращается к серверу, на котором хранится база данных.
В свою очередь сервер возвращет числовой адрес сайта. Также DNS-сервер кэширует данные об IP-адресах, которые пользователь открывал ранее,
чтобы сократить время поиска при повторных обращениях.

HTTP (Hyper Text Transfer Protocol) - протокол передачи данных по сети. Обеспечивает передачу любого формата данных. Это документ, который описывает правила взаимодействия клиента и сервера.
HTTPS (Hyper Text Transfer Protocol Secure) - защищенный протокол передачи данных. В отличие от HTTP данные перед отправкой на сервер шифруются, а затем дешифруются сервером.
Например, чтобы защитить информацию о вводимых данных банковской карты.
Версии HTTP:
HTTP/0.9 - разработан в 1989 году, упорядочивал правила взаимодействия между клиентами и серверами HTTP, поддерживал всего 1 метод GET.
HTTP/1.0 - разработан в 1996 году, поддерживал методы: GET, HEAD и POST
HTTP/1.1 - разработан в 1997 году, поддерживал 7 методов: GET, POST, PUT, DELETE, HEAD, OPTIONS, TRACE, в 2010 году был добавлен метод PATCH, а в 2014 — CONNECT, TCP-соединение может оставаться открытым после отправки ответа на запрос, что позволяет посылать несколько запросов за одно соединение, стала возможной более простая организация виртуального хостинга.
HTTP/2 - выпущен в 2015 году, имеет новый формат данных, не имеющих обратной совместимости с HTTP/1.x, более эффективное использование сетевых ресурсов и уменьшение задержек, передача нескльких потоков по одному каналу, расстановка приоритетов для запросов, сжатие заголовков, загрузка нескольких элементов параллельно при использовании одного TCP-соединения.
HTTP/3 - поддержка частью сервисов была добавлена в 2019 году, использует протокол UDP вместо TCP, в данный момент около 20% сайтов доступны по этой версии протокола, оптимизирована передача пакетов, более эффективное шифрование данных
